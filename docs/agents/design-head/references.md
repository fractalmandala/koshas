# Design Head — Core-doc References

## Reading Order

1. **`docs/core-docs/koshas-specs-v1.md`** — Understand the product: all sheaths, content types, search, organization.
2. **`docs/core-docs/design-principles.md`** — Ground yourself in the design philosophy before starting any work.
3. **`docs/core-docs/tooling.md`** — Find reference images (Assets section) and skills to use.
4. **`docs/core-docs/team.md`** — Understand handoff points to Interaction Engineer.
5. **`docs/core-docs/architecture-decisions.md`** — Understand tech constraints that affect design (e.g., local-first, editor stack).
