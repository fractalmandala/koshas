# Product Lead — Core-doc References

## Reading Order

1. **`docs/orchestration.md`** — Always read first. Know current state, milestone, blockers.
2. **`docs/milestones.md`** — When reviewing milestone scope or planning next steps.
3. **`docs/tasks.md`** — When delegating or checking task status.
4. **`docs/core-docs/koshas-plan-v1.md`** — When reviewing roadmap or release criteria.
5. **`docs/core-docs/koshas-specs-v1.md`** — When verifying feature scope or spec accuracy.
6. **`docs/core-docs/tooling.md`** — When deciding if a new skill or tool is needed.
7. **`docs/core-docs/team.md`** — When assigning tasks or resolving ownership questions.
8. **`docs/core-docs/design-principles.md`** — When evaluating feature proposals against principles.
9. **`docs/core-docs/architecture-decisions.md`** — When tech decisions need review.

## What to Update

| File | When to update |
|---|---|
| `docs/orchestration.md` | Every state change — task completion, milestone progress, blockers |
| `docs/milestones.md` | When milestone scope changes or a milestone ships |
| `docs/tasks.md` | When tasks are added, removed, or change scope |
| `docs/core-docs/koshas-plan-v1.md` | When the roadmap shifts |
| `docs/core-docs/koshas-specs-v1.md` | When a feature change during development affects the spec |
